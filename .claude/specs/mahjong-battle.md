# Mahjong Battle Spec

1인 전투형 리치마작 시스템. 씬: `MahjongBattleScene`. 빌더: `Tools > Build MahjongBattle Scene`.

## 핵심 규칙

- 표준 136장 패산. 1인이 모든 패산 소비. 호출 없음(치·퐁·명깡 없음). 왕패 없음.
- 라운드 시작 시 도라 인디케이터 5장 별도 보관. 나머지 131장이 쯔모 대상.
- 손패 13장 + 쯔모 1장 = 14장. 안깡만 가능. 깡 시 패산 끝에서 영상수 보충.

## 턴 구조 (1차 임시)

- 플레이어 1쯔모 → 1버림 반복.
- 버림 즉시 모든 적에게 `EnemyMahjongState.OnPlayerDiscard` 통지. 일치 시 적이 "론!"으로 공격.
- 버림/론 처리 후 생존 적마다 2% 확률로 `EnemyMahjongState.TryTsumo` 판정. 성공 시 해당 적이 "쯔모!"로 자동 공격.
- 발동 시 적별 데미지 → 플레이어 하트 감소. 전체 턴 후 적 조합 Reroll.

## 데미지 테이블 (수리 기반)

전체 역 완성 = 100% AOE. 중간 포기 = 50% AOE.

야쿠만 = 16 절반하트(= 5하트 풀 킬 1대 기준).
일반 화료: 1한=1, 2한=2, 3한=3, 4한=4, 5한=4(満貫), 6~7한=6(跳満), 8~10한=8(倍満), 11~12한=12(三倍満), 13+한=16(카운트 야쿠만).
도라는 `TotalHan`에 합산 → 같은 테이블로 룩업.

중간 포기: `(멘츠 × 0.5 + 머리 × 0.25 + 깡 × 0.75) × 0.5`, `ceil`. 0 이상 절반하트 단위.

상수는 모두 `MahjongDamageTable.cs` 단일 파일.

## 야쿠 범위

표준 리치마작 야쿠 대부분 인정. 평가기: `YakuEvaluator.cs`.
미구현(1차): 一発, 자풍/장풍, 天和/地和, 海底/河底, 嶺上開花, 槍槓, 우라도라.

## 적 규칙 (1~3랭크)

- 스폰 시 "한 장만 더 있으면 완성되는 조합" 2개. 슌쯔(멘츠)/커쯔(멘츠)/또이츠(머리) 중 40/40/20 분포. 플레이어 자원과 완전 독립.
- 발동 데미지(절반하트 단위): 슌쯔 = rank × 0.5, 커쯔 = rank × 1.0, 또이츠 = rank × 0.25, 도라 1장당 추가 rank × 1.0. `ceil`.
- 발동 후·플레이어 공격 후·중간 포기 후 모두 Reroll.
- 대기패 정보 공개: 1~2성 적은 위험패 1장을 항상 표시, 3성 적은 매 턴 5% 확률로 위험패 1장을 표시, 4~5성 적은 실제 패를 표시하지 않음.
- 4·5랭크 전투는 향후 별도 설계. `EnemyMahjongState`는 rank 필드 유지하여 확장 여지 확보.

## 전투 로그

- 시작 시 `적과 조우했다!` 또는 보스전 `강적과 조우했다!`와 적 목록을 하나의 메시지로 묶어 표시.
- 적 발동, 데미지, 화료/중간 포기, 리치/안깡, 직감, 승패처럼 결과 해석에 필요한 정보만 표시.
- 도라 공개 수, 버림패, 타겟 변경처럼 화면 UI로 이미 확인 가능한 단순 상태는 로그에 표시하지 않음.

## 미확정 (튜닝 단계에서 결정)

- 드로우/버림 세부 턴 구조
- 전체 역 데미지 수치 최종 스케일
- 리치/깡 버튼의 상세 기능(현재 리치=+1한, 깡=안깡+영상수만)

## 검증

- EditMode 테스트: `Window > General > Test Runner > EditMode`. 패산/분해/야쿠/데미지/적 풀 단위 테스트가 `Assets/Editor/Tests/Mahjong/`.
- Play 검증: 마작 캐릭터 선택 → ExploreScene → 전투 진입 시 `MahjongBattleScene` 로드.
